Case ID 用例标题 HTTP 方法 接口路径 测试目标 前置条件 预期结果
TC01查询事项任务列表返回分页数据POST/task/query验证 /task/query 能够返回分页统计信息与 rows 列表。已获取有效 Authorization；系统中存在至少一条事项任务。响应 200 且包含 total/pageNum/pageSize 字段以及 rows 数组。
TC02查询事项任务详情GET/task/detail验证 /task/detail 可返回指定任务完整信息。存在可访问的任务 ID。响应 200，data.id 与请求一致且包含 executes 列表。
TC03查询事项任务详情（无子任务树）GET/task/detail/noTree验证 /task/detail/noTree 返回任务基础信息。存在可访问的任务 ID。响应 200，返回 AjaxResult 格式且 data 字段存在。
TC04查询任务执行人与审批人信息GET/task/queryExecute验证 /task/queryExecute 返回执行人列表。任务存在，且已分配执行人。响应 200，data 为执行人/审批人信息数组。
TC05执行记录反查任务GET/task/executeIdToTaskId验证 /task/executeIdToTaskId 可通过执行记录 ID 获取任务 ID。任务存在执行记录。响应 200 且 data 等于目标任务 ID。
TC06催办任务POST/task/urge验证 /task/urge 可向执行人发送催办请求。任务处于可催办状态，存在执行人。响应 200，返回成功消息。
TC07更新任务基本信息POST/task/update验证 /task/update 能成功提交任务信息，即使内容未改动。任务详情可查询且具有必填字段。响应 200，AjaxResult 表示更新成功。
TC08查询任务模板列表GET/task/template/query验证 /task/template/query 返回模板集合。系统中存在至少一个任务模板。响应 200，data 为模板列表。
TC09新增任务模板POST/task/template/add验证 /task/template/add 可基于已有模板新增条目。具备模板示例用于复制生成新模板。响应 200，新增模板可在后续查询中检索到。
TC10更新任务模板POST/task/template/update验证 /task/template/update 能修改模板名称等基础信息。存在可编辑模板。响应 200，返回成功消息。
TC11删除任务模板POST/task/template/delete验证 /task/template/delete 可移除指定模板。存在可删除的模板 ID。响应 200，模板自列表中移除。
TC12重新打开任务POST/task/open验证 /task/open 可将任务状态恢复为进行中。任务存在且可被重新打开。响应 200，AjaxResult 表示操作成功。
TC13关闭任务后恢复POST/task/close验证 /task/close 能关闭任务且事后可以重新打开。任务存在并允许关闭。响应 200，随后通过 /task/open 可恢复状态。
TC14完结任务后恢复POST/task/end验证 /task/end 能完结任务且可重新打开。任务存在并允许完结。响应 200，随后通过 /task/open 可恢复执行。
TC15完成任务后恢复POST/task/complete验证 /task/complete 能标记完成并可重新激活任务。任务存在并允许完成。响应 200，随后通过 /task/open 可恢复执行。
TC16审批任务通过POST/task/audit验证 /task/audit 能提交审批通过操作。任务存在待审批记录，当前用户具备审批权限。响应 200，返回成功消息。
TC17撤销审批结果POST/task/audit/cancel验证 /task/audit/cancel 能撤销上一审批动作。任务存在可撤销的审批记录。响应 200，返回成功消息。
TC18执行人提交执行记录POST/task/execute/submit验证 /task/execute/submit 可提交执行结果（若条件不足，将返回业务校验提示）。任务存在执行记录且当前用户为执行人。响应 200 或 400 且结构符合 AjaxResult。
TC19撤回执行提交POST/task/execute/submit/revoke验证 /task/execute/submit/revoke 可撤回指定执行提交。执行记录存在且允许撤回。响应 200，返回成功消息。
TC20保存执行备注与附件POST/task/execute/attachAndComment/save验证 /task/execute/attachAndComment/save 能提交执行说明（若缺少必填附件可能返回业务提示）。执行记录存在。响应 200 或 400 且结构符合 AjaxResult。
TC21更换执行人POST/task/execute/change验证 /task/execute/change 可将执行人切换为其他用户。任务至少有两个执行人供切换。响应 200，并可恢复为原执行人。
TC22邀请新执行人POST/task/execute/invite验证 /task/execute/invite 可新增执行人。任务存在，可邀请的用户具备执行权限。响应 200，返回成功消息。
TC23移除执行人并恢复POST/task/execute/remove验证 /task/execute/remove 可移除执行人且事后可重新邀请。任务存在多个执行人。响应 200，随后重新邀请成功。
TC24新增抄送人POST/task/copyer/add验证 /task/copyer/add 可添加抄送用户。任务存在；提供的用户可以作为抄送人。响应 200，返回成功消息。
TC25创建任务接口的必填校验POST/task/add验证 /task/add 对缺失信息的校验逻辑。提供最小化任务 payload；部分字段缺失。响应 200 或 400，但返回结构符合 AjaxResult。
